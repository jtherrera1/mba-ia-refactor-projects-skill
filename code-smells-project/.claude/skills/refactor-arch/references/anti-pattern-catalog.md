
This documentation consolidates some **anti-patterns** that can help identify them in different projects.

---

## Severity classification for anti-patterns

To standardize your audit and the reports generated by the AI, use the following classification scale based on MVC and SOLID issues:

CRITICAL: Serious architecture or security failures that prevent proper functioning, expose sensitive data (e.g., hardcoded credentials, SQL Injection), or completely violate separation of responsibilities (e.g., a “God Class” containing database access, complex business logic, and routing in the same file).

HIGH: Major violations of the MVC pattern or SOLID principles that make maintenance and testing very difficult (e.g., heavy business logic trapped inside Controllers, strong coupling without Dependency Injection, or use of mutable global state across the entire application).

MEDIUM: Standardization issues, code duplication, or moderate performance bottlenecks (e.g., N+1 database queries, improper middleware usage, missing route validations).

LOW: Readability improvements, poor variable naming, or loose “magic numbers” scattered throughout the code.

# Detailed Anti-Patterns

## 1. God Class - CRITICAL
**What it is:** A single class concentrates multiple responsibilities.

**Where it appears:** Classes like `AppManager`, `SystemManager`, `MainService`.

**Problems:**
- Hard to test
- High coupling
- Risky changes

**How to fix:**
- Split into Services, Controllers, and Repositories
- Apply SRP (Single Responsibility Principle)

---

## 2. SQL Injection - CRITICAL
**What it is:** Building SQL queries via text concatenation.

**Where it appears:**
```python
"SELECT * FROM users WHERE email='" + email + "'"
```

**Problems:**
- Data leakage
- Table deletion
- Full system compromise

**How to fix:**
- Prepared Statements
- Secure ORM
- Parameter binding

---

## 3. Hardcoded Secrets - CRITICAL
**What it is:** Keys, tokens, or passwords embedded in source code.

**Where it appears:**
```js
apiKey = "prod_secret_key"
```

**Problems:**
- Leaked via Git
- Compromises production

**How to fix:**
- Environment variables
- Secret Manager

---

## 4. Plain Text Password - CRITICAL
**What it is:** Passwords stored without encryption/hashing.

**Where it appears:** Legacy user database.

**How to fix:**
- bcrypt
- Argon2
- Salt + secure hash

---

## 5. Broken Authentication - CRITICAL
**What it is:** Public administrative endpoints.

**Where it appears:**
- `/admin/query`
- `/financial-report`

**How to fix:**
- JWT / Session Auth
- RBAC
- Authorization middleware

---

## 6. Callback Hell - HIGH
**What it is:** Many nested callbacks.

**Where it appears:** Legacy Node.js.

**How to fix:**
- async/await
- Promises
- Decoupled services

---

## 7. N+1 Query Problem - MEDIUM
**What it is:** For each item, a new database query is made.

**Problems:** Poor performance.

**How to fix:**
- JOIN
- Eager loading
- Batch queries

---

## 8. Duplicate Code - MEDIUM
**What it is:** Same validation repeated across multiple files.

**How to fix:**
- Reusable functions
- Helpers
- Centralized validators

---

## 9. Print Logging - LOW
**What it is:** Use of `print()` in production.

**How to fix:**
- Winston
- Python logging
- Structured logs

---

## 10. Magic Numbers - LOW
**What it is:** Values without context.

```js
if(timeout > 200)
```

**How to fix:**
- Named constants
- External configuration

---

## 11. Orphan Data - MEDIUM
**What it is:** Deletion of records without cascade.

**How to fix:**
- Foreign Keys
- ON DELETE CASCADE
- Soft delete

---

## 12. Tight Coupling - HIGH
**What it is:** Classes depend directly on each other.

**How to fix:**
- Dependency Injection
- Interfaces

---

## 13. Fat Controller - HIGH
**What it is:** Controller containing business logic.

**How to fix:**
- Move rules to Service Layer

---

## 14. Global Mutable State - HIGH
**What it is:** Global variables modified by multiple parts of the code.

**How to fix:**
- Immutability
- Isolated contexts
- Dependency injection

---

## 15. Deprecated/Obsolete APIs - MEDIUM
**What it is:** Use of deprecated or obsolete functions, modules, libraries, or endpoints that are no longer maintained or recommended.

**Where it appears:**
```python
import cgi  # deprecated since Python 3.11
os.popen("ls")  # replaced by subprocess
from distutils.core import setup  # replaced by setuptools
```
```js
new Buffer(data)  // replaced by Buffer.from()
fs.exists(path, cb)  // replaced by fs.access()
```

**Problems:**
- Security vulnerabilities in unmaintained code
- Removal in future language/framework versions
- Missing bug fixes and performance improvements
- Compatibility issues with newer dependencies

**How to fix:**
- Replace with modern equivalents recommended by official documentation
- Use linters/deprecation warnings (e.g., `pylint`, `eslint-plugin-deprecation`)
- Keep dependencies up to date
- Monitor changelogs for deprecation notices
